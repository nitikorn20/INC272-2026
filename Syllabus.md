# INC272: Web-Based IoT Applications

## Course Overview

This course focuses on the design and implementation of browser-based monitoring and control applications for IoT-oriented workflows. Students use HTML, CSS, and JavaScript to build interactive web interfaces, work with a Node.js mock hardware simulator through WebSockets, and complete a project-oriented learning sequence that ends with a group final project.

The course is intentionally designed to keep setup simple. Instead of depending on `Proteus`, `VSPD`, or other installer-heavy simulation workflows, the default path uses a lightweight local mock simulator so students can focus on application behavior, UI logic, and project delivery.

This is a project-oriented course. The heaviest assessment weight is intentionally placed on the group final project, while the readiness task and the individual exam are used to confirm that each student can work independently and understands the required foundation.

## Learning Outcomes

Upon successful completion of this course, students should be able to:

1. Explain the structure of a browser-based IoT application.
2. Build small web interfaces with HTML, CSS, and JavaScript.
3. Implement DOM manipulation and event-driven interaction logic.
4. Connect browser interfaces to simulated device-like data and behaviors.
5. Design and deliver a group project that demonstrates a complete browser-based monitoring or control workflow.

### Readiness Task

The readiness task is completed individually during Weeks 01-02 and checked no later than Week 03. Students are expected to:

- install the required tools
- run the mock hardware server
- open the assigned example applications
- prepare and submit simple evidence of successful setup

If a student still cannot run the required workflow by the Week 03 onsite check, the readiness task receives `0`. Support may still be provided so the student can continue into the exam and project stages.

### Individual Exam

The exam is conducted individually during Week 04 and is `paper-based only`. It is designed to measure actual understanding without relying on live coding tools or AI assistance.

The exam scope is limited to the assigned materials in this repository, with the strongest emphasis on:

- HTML basics
- CSS basics
- JavaScript basics
- DOM and event logic
- reading and understanding the assigned example applications
- a small amount of browser-to-simulator interaction understanding

See [Exam Blueprint](./docs/exam-blueprint.md) for the topic distribution and expected question style.

### Group Final Project

The final project is completed in groups and is treated as one main assessment category. It is introduced in Week 03, scoped in Week 04, built in Week 05, and presented in Week 06.

The project category may internally consider:

- fit to the project brief
- scope confirmation
- individual Q&A
- implementation quality
- presentation and demo quality
- repository and documentation quality

## Policies

### Readiness Task Policy

- readiness evidence must be completed before or during the Week 03 readiness check
- students who are not ready by the Week 03 check receive `0` for the readiness task
- instructor support may still be provided after the score decision

### Exam Attendance and Make-up Policy

- the Week 04 exam is conducted onsite according to the scheduled class date and time
- students who miss the exam without a valid reason receive `0`
- students who miss the exam with a valid documented reason may request a make-up exam
- make-up exam scores are capped at `70%` of the earned score

### Submission Policy

- students are expected to complete required weekly work on time
- late readiness submissions are not accepted after the Week 03 readiness check
- late final project submissions may be treated as not submitted unless approval is granted or valid documented evidence is provided

### Academic Expectation

- the readiness task and exam are individual work
- the final project is group work
- students are expected to understand and explain the work submitted under their name or group
