# Week 03 Readiness Checklist

## Purpose

Use this checklist during the first onsite session to verify that each student is technically ready for the rest of the course.

* * *

## Scoring Rule

- if the required workflow is ready by the Week 03 check, award the `5%` readiness task score
- if the required workflow is still not ready by the Week 03 check, assign `0` for the readiness task
- support may still be provided after the score decision so the student can continue to the exam and project stages

* * *

## Quick Check Sequence

Each student should be able to show the following on their own machine:

1. `node -v` and `npm -v` run successfully.
2. `npm start` works in `simulator/mock-hardware-server`.
3. A static file server or Live Server is working.
4. `examples/apps/web_apps/web01_getting_started/index.html` opens without browser errors.
5. At least one interaction example works, such as `web02_buttons_led_control` or `web03_psw_led_indicators`.
6. At least one data-display example works, such as `web04_adc_gauges` or `web05_adc_plotters`.

* * *

## Suggested Evidence to Observe

- version output in the terminal
- the mock server running normally
- the example page loading locally
- changing UI state or changing data values in the browser

* * *

## Fast Troubleshooting Priorities

If a machine is not ready, check these first:

- Node.js installation
- whether the mock server is running
- whether the correct project folder is open
- whether the page is opened through a static server instead of direct file opening
- whether the browser console shows a simple path or script error
