# Release Checklist

## Purpose

Use this checklist before publishing the repository to students.

* * *

## Repository Readiness

- confirm that the branch is up to date
- confirm that the working tree is clean
- confirm that `README.md`, `Syllabus.md`, and `Outline.md` still match the intended course model
- confirm that Week 01 to Week 06 documents still match the planned teaching schedule

* * *

## Course Content Readiness

- confirm the course title, academic year, and schedule are correct
- confirm the assessment model is correct
- confirm the exam format and exam blueprint are correct
- confirm the final project guide and Week 03 briefing materials are correct
- confirm the readiness task policy is still the intended policy

* * *

## Technical Readiness

- run the smoke tests in `tools/example-smoke-test`
- verify that the mock hardware server starts correctly
- verify that the assigned Week 01 and Week 02 examples still work
- confirm that setup guides for Windows and macOS still match the intended workflow

* * *

## Student-Facing Documents

- confirm that all major documents are linked from `README.md`
- confirm that `Student Quick Start` still reflects the fastest student workflow
- confirm that the Week 03 readiness checklist is ready for onsite use
- confirm that legacy materials are clearly marked as support only

* * *

## Publishing Steps

1. push the latest `main` branch to the remote repository
2. confirm that the remote default branch is `main`
3. review the repository on GitHub after push
4. verify that key documents render correctly in the browser
5. publish or share the repository link with students when ready
