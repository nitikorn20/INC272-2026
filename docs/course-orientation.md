## What This Course Is About

This course focuses on building browser-based monitoring and control applications with:

- HTML
- CSS
- JavaScript
- a Node.js mock WebSocket-based hardware simulator

The main goal is not circuit simulation. The main goal is to build usable browser applications that can display device-like data, react to events, and support a simple monitoring or control workflow.

* * *

## Default Technical Workflow

Students are expected to:

1. study the basic web materials
2. run the assigned examples
3. run the mock simulator locally
4. build a group project using the same lightweight workflow

Required tools:

- Node.js
- Visual Studio Code
- modern browser

Not required in the default path:

- Proteus
- Virtual Serial Port Driver

* * *

## What Students Should Finish Before Week 03

Before the first onsite session, students should already be able to:

- run `node -v` and `npm -v`
- run the mock hardware server
- open the assigned starter web apps
- show simple evidence of the readiness task
