# Exam Blueprint

## Exam Identity

| Item | Information |
|------|-------------|
| Assessment | Individual Exam |
| Week | `Week 04` |
| Format | `Paper-based only` |
| Duration | announced by the instructor |
| Scope | limited to the assigned repository materials |

* * *

## Exam Purpose

This exam is used to measure individual understanding of the core browser-side concepts required for the course project.

The exam is intentionally paper-based so that students are evaluated on understanding rather than on access to coding tools or AI assistance.

* * *

## Blueprint by Topic

| Topic Area | Suggested Weight | Main Focus |
|------------|------------------|------------|
| HTML basics | `20-25%` | structure, tags, attributes, page meaning |
| CSS basics | `20-25%` | selectors, styling, layout basics, visual interpretation |
| JavaScript basics | `25-30%` | variables, conditions, functions, simple logic |
| DOM and events | `20-25%` | interaction flow, element updates, event handling |
| Simulator-related understanding | `5-10%` | simple browser-to-simulator workflow understanding |

The largest share should remain on `HTML`, `CSS`, and `JavaScript` basics.

* * *

* * *

## Expected Question Styles

- read and interpret short code snippets
- identify or fix small mistakes in code fragments
- explain what a given HTML, CSS, or JavaScript snippet will do
- explain DOM or event outcomes
- explain a small browser-to-simulator interaction scenario

* * *

## What Will Not Be Emphasized

- advanced backend topics
- heavy simulator internals
- tooling commands beyond the basic assigned workflow
- topics outside the assigned examples and weekly materials

* * *

## Preparation Advice

- review the assigned examples, not random outside tutorials
- focus on understanding what the code does and why
- practice reading and fixing small snippets on paper
- review how the starter apps connect browser behavior to simulator responses
