# Course Workflow Reference

## Default Course Model

- Week 01: online web foundations
- Week 02: online simulator familiarization
- Week 03: onsite readiness check and project briefing
- Week 04: onsite individual exam and project scope confirmation
- Week 05: onsite project implementation and checkpoint
- Week 06: onsite final project demo

## Default Technical Model

- browser-based teaching stack
- static files for examples
- Node.js mock hardware server for simulated device behavior
- no default dependence on Proteus, VSPD, or Windows-only simulation setup

## Default Assessment Model

- Readiness Task: 5%
- Exam: 25%
- Final Project: 70%
- Exam format: paper-based only
- Readiness rule: if the required workflow is still not ready by the Week 03 check, score the readiness task as 0

## Maintenance Heuristics

- Prefer editing docs before adding more tooling.
- Prefer small, clear student instructions over long explanations.
- Reuse the legacy examples when they still match the teaching goal.
- If a new feature makes setup heavier, question whether it belongs in the main path or an optional track.
